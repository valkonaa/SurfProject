# Этап 2: Архитектурный план и схема данных

**Цель:** Спроектировать легковесную архитектуру и структуру хранения данных (in-memory для MVP).

**Промпт для ИИ:**
> "Действуй как Solution Architect. Напиши схему данных и структуру API для FastAPI приложения 'Гончарная мастерская' на основе 3 фич: просмотр мастер-классов, создание бронирования, админка бронирований."

### Схема данных (Сущности)

1. **Workshop (Мастер-класс)**
   - `id`: int
   - `title`: str (название, например, "Лепка кружки из глины")
   - `price`: int (цена в рублях)
   - `slots`: int (оставшиеся свободные места)

2. **Booking (Бронирование)**
   - `id`: int
   - `workshop_id`: int
   - `client_name`: str
   - `client_phone`: str

### Спецификация API (Эндпоинты)
* `GET /workshops` — Получить список всех мастер-классов.
* `POST /bookings` — Создать бронь (принимает `workshop_id`, `client_name`, `client_phone`).
* `GET /admin/bookings` — Посмотреть все бронирования.
