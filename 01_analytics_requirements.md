# Этап 1: Анализ и требования к MVP — «Гончарная мастерская»

**Цель:** Сформировать требования к MVP e-commerce платформы для записи на мастер-классы по гончарному делу и покупке изделий.

**Промпт для ИИ:**
> "Действуй как системный аналитик. Сгенерируй MVP требования (User Stories и основные сценарии) для веб-приложения 'Гончарная мастерская' (несложный e-commerce). Выдели 3 ключевые фичи для реализации."

### User Stories
1. **Как пользователь**, я хочу видеть список доступных мастер-классов, чтобы выбрать подходящий по времени и теме.
2. **Как пользователь**, я хочу оформить заказ (запись) на мастер-класс, чтобы забронировать место.
3. **Как администратор**, я хочу просматривать список всех бронирований, чтобы контролировать загрузку мастерской.

### Основные сценарии (Use Cases)
* **Сценарий 1: Просмотр и бронирование.** Пользователь заходит на сайт -> видит список мастер-классов -> нажимает "Забронировать" -> вводит имя и телефон -> получает подтверждение.
* **Сценарий 2: Просмотр админом.** Администратор запрашивает список броней -> система выдает актуальный перечень с контактами клиентов.

### Фичи для реализации:
1. Вывод каталога мастер-классов (`GET /workshops`).
2. Создание бронирования (`POST /bookings`).
3. Просмотр списка бронирований для админа (`GET /admin/bookings`).
